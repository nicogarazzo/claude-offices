Core Concepts

Three building blocks

Based on the Soul + Skills architecture from Hermes Agent (Nous Research). The Soul gives each agent a persistent identity modeled after real domain experts. Skills act as office dispatchers that orchestrate the team.

◆

Office

A team of agents organized around a domain. Each office is a Claude Code Skill that acts as a dispatcher, routing tasks to the right agent and orchestrating the quality pipeline.

◉

Soul

Each agent has a persistent identity: personality, voice, principles, and domain expertise. The Soul defines WHO the agent is, not just what it does. It travels with the agent across every invocation.

⇄

Two Modes

Every agent operates in two modes: Production (create/transform work) and Critique (evaluate work from their specialty). This duality enables the iterative quality pipeline.

Quality Pipeline

Every office runs 2 rounds

No deliverable leaves an office without passing through at least 2 production rounds and a cross-critique session. This is the standard contract for every office, regardless of domain.

Round 1: Production
↓
Agent A produces
↓
Agent B produces
↓
Agent C produces
↓
Cross-Critique Table (parallel)
↓
Agent A critiques
Agent B critiques
Agent C critiques
↓
Round 2: Rewrite with feedback
↓
Agent A rewrites
↓
Agent B refines
↓
Agent C: Final Verdict
Quick Start

Up and running in 3 commands

1. Clone and install

git clone https://github.com/nicogarazzo/claude-offices.git
cd claude-offices
./swarm install

2. Create your first office

# Using the CLI
./swarm new-office oficina-research "Deep research and analysis team"
./swarm add-agent oficina-research analyst "Primary research and data gathering"
./swarm add-agent oficina-research synthesizer "Connects findings into insights"
./swarm add-agent oficina-research critic "Validates methodology and conclusions"

# Edit each agent's .md file to customize the Soul
# Then install
./swarm install

3. Or use natural language

# In any Claude Code session, after installing:
/oficina-creator

"Create a UX research office with a user researcher,
 a wireframer, and a usability critic"
ⓘ After ./swarm install, restart Claude Code so it detects the new skills and agents. Your offices will appear as available skills (e.g., /oficina-research).
CLI Reference

The swarm command

new-office

./swarm new-office <name> "description"

Creates a new office directory with a SKILL.md template that includes the 2-round pipeline protocol.

add-agent

./swarm add-agent <office> <agent-name> "description"

Adds an agent to an office with both Production and Critique modes scaffolded. Auto-updates the office roster and allowed-tools.

remove-agent

./swarm remove-agent <office> <agent-name>

Removes an agent from an office and updates the roster.

list

./swarm list

Shows all offices, their agents, descriptions, and installation status.

install

./swarm install

Copies all offices to ~/.claude/skills/ and ~/.claude/agents/. Also installs core skills like oficina-creator.

uninstall

./swarm uninstall

Removes all installed offices from ~/.claude/.

sync

./swarm sync

Updates the routing table in Claude's project memory so it knows which office to suggest for each task type.

Natural Language Creator

Describe an office, get a team of experts

The /oficina-creator skill lets you describe an office in plain language. It researches the best domain experts to model each agent's Soul, presents them for your approval, then generates everything.

What you say

"Create an office for translating technical documents from English to Spanish"

What it researches

Before generating anything, the skill finds the best figures for each role:

  • Eugene Nida as first translator: dynamic equivalence theory, meaning over literality
  • Edward Sapir as cultural reviewer: Sapir-Whorf hypothesis, language shapes thought
  • Umberto Eco as final editor: "Saying Almost the Same Thing", translation as negotiation

Each figure is selected for their documented methodology, not just fame. Alternatives are considered and the selection is presented before generating.

What you get

A complete oficina-traduccion with:

  • Agent nida with a Soul encoding dynamic equivalence principles
  • Agent sapir with a Soul encoding cultural-linguistic review methodology
  • Agent eco with a Soul encoding semiotic precision and editorial negotiation
  • Each with Production mode, Critique mode, and structured delivery templates
  • 2-round pipeline: Nida translates → Sapir reviews culture → Eco approves. Then all 3 critique in parallel. Then Round 2 with feedback.
Example

The Copywriting Office

A complete B2B copywriting pipeline that ships with the project as a reference implementation.

Jordan Belfort
First Draft Writer
High-energy sales copy. Writes with urgency, focuses on pain points, uses frameworks like AIDA and PAS. In critique mode, evaluates sales force, urgency, and emotional connection.
Emma Stratton
Punchy Filter
Anti-jargon, anti-fluff editor. Kills corporate speak, enforces short sentences, demands specificity. In critique mode, scores punch, clarity, and residual jargon.
Ogilvy
Final Reviewer
Precision and elegance. Validates claims, checks persuasive structure, ensures the copy sells without feeling like it sells. Issues the final verdict: APPROVED or REQUIRES REWRITE.

Pipeline flow

Round 1: Jordan drafts with maximum energy. Emma strips jargon and tightens rhythm. Ogilvy reviews structure and passes to critique.

Critique: All three evaluate the Round 1 output in parallel. Jordan flags lost sales energy. Emma catches surviving jargon. Ogilvy checks claims and persuasive arc.

Round 2: Jordan rewrites incorporating all three critiques. Emma re-filters. Ogilvy delivers the final approved version.
